Создаём базовый шаблон HTML 5


Чтобы обозначить тот или иной тег на странице, к демонстрации мы подключим подсветку тегов. 
Для этого используем служебный стилевой файл outline.css. 

В шапке страницы появился ещё один служебный тег — <base>, 
с помощью него мы сократим путь до используемых ресурсов. 

В качестве значения атрибута href укажем базовый адрес (URL) 
полного пути, по которому расположены подключаемые ресурсы. 

Подробнее о теге <base> можно прочитать в спецификации.

В блоке main выделяются три раздела: главная новость, обзоры и промо-блок. Эти разделы неразрывно связаны с тематикой сайта, 
поэтому размечаем их элементами <section>. Добавим атрибут class для разделов со значениями: headline — главная новость, 
сенсация, reviews — обзоры и promo — промо-блок. Подвал в данном случае содержит простые элементы, и крупные блоки для разметки 
не понадобятся.

Приступаем к добавлению заголовков для всей страницы и для смысловых разделов, выделенных на предыдущем шаге. Макет не содержит 
заголовок, который подошёл бы для всей страницы. Добавим его. Поскольку это главная страница сайта о культурных событиях, для неё 
вполне подойдёт заголовок «Новости культуры». Так мы чётко обозначили тематику сайта. Этот заголовок отсутствует на макете, добавим 
ему атрибут class="visually-hidden" для того, чтобы доступно его скрыть. Подробно узнать о том, как правильно прятать, можно 
в HTML-шорте «Как прятать». У блока с главной новостью есть свой заголовок — «Подводный отель», используем его. Это будет заголовок 
второго уровня. Раздел с обзорами и раздел промо-блока имеют свои заголовки на макете, добавим их в разметку.

Оглавление нашего документа готово. Можем приступать к разметке мелких элементов. Добавим логотип в главную навигацию сайта. 
Обратите внимание на логотип сайта — «Новости культуры». Этот текст написан нестандартным шрифтом. Такой шрифт соответствует дизайн-макету 
компании. Чтобы не было искажения фирменного стиля в ситуации, когда на компьютере пользователя отсутствует необходимый шрифт, 
текст преобразуют в изображение. Получаем в качестве логотипа изображение. Для тега <img> заполним атрибут src. Не забываем указывать 
размеры и альтернативный текст, то есть атрибуты width, height и alt.

Изображения для этой демонстрации находятся на специальном сервере, там же где расположен служебный стилевой файл outline.css. 
Так как мы используем тег <base>, будем обращаться к картинкам по относительному адресу, вот так — img/logo.svg. 
При работе над разметкой на вашем компьютере вы также можете использовать относительные адреса. Подробнее об адресах рассказано в 
интерактивном курсе про изображения.

Предусмотрим переход на главную страницу при нажатии на логотип сайта. Этот переход понадобится на других страницах сайта. 
Обернём изображение тегом <a>. Поскольку в данный момент мы находимся на главной странице, используем тег <a> без атрибута href, 
чтобы наша ссылка не была активна.

Навигация по сайту — это список однотипных и одноранговых элементов-ссылок. При разметке навигации используем тег <ul>. 
Для списка и элементов списка укажем атрибут class со значениями site-navigation и site-navigation-item соответственно.

Переходим к блоку с главной новостью. Добавим в блок изображение отеля. Указываем источник для изображения. Не забываем про размеры и 
альтернативный текст.

Текст новости разметим тегом <p>

Заголовок главной новости должен перекидывать на страницу с полным перечнем новостей. Поэтому обернём текст заголовка тегом <a>. 
В качестве атрибута href мы поставили заглушку, то есть указали символ #. Сейчас эта ссылка ведёт на несуществующую страницу. 
С помощью заглушки мы избавились от возможности получить ошибку при переходе по ссылке.

Переходим к разделу «Обзоры». Обзоры — это список однотипных элементов. Добавим элемент <ul> в разметку и подготовим элементы списка. 
Добавим атрибут class для списка и отдельных элементов с обзорами.
